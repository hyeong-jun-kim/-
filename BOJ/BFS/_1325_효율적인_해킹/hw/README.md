## Info
<a href="https://www.acmicpc.net/problem/1325" rel="nofollow">1325_효율적인_해킹</a>

## ❗ 풀이 방법
1. main에서 입력받고 solution으로 넘어가기
2. solution에서 1번노드(컴퓨터) 부터 쭉 탐색
3. 현재 노드 queue에 넣고 방문처리
4. bfs 진행
5. 해킹한 컴퓨터 갯수 셀 변수 count 선언(현재 노드 1개 들어가있으니 1로 초기화)
6. 연결정보를 토대로 bfs 진행. 현재 노드를 신뢰하는 컴퓨터가 있다면 count++

7. count를 return 하고 최댓값 갱신, 기록배열에 count 수 기록

8. 기록 배열에서 최댓값인 idx(컴퓨터번호) answer 배열에 담기

## 🙂 새로 알게된 점

* 푸는건 쉬웠는데 메모리와 시간이 너무 깐깐했다.이런문제가 제일 싫다

### 메모리초과 내용
1. 컴퓨터번호 : 해킹할 수 있는 갯수 Map 선언하여 풀었더니 메모리초과 -> 배열로 변경
2. 연결정보를 2차원 배열로 받았더니 메모리초과 -> ArrayList로 연결정보 받는것으로 변경

### 시간초과 내용
1. bfs 함수 내에서 visited 배열 쭉 탐색하여 컴퓨터 갯수 세는것으로 하였더니 시간초과
2. Scanner 입력 시간초과

- Scanner 입력 시간초과는 앞으로도 고칠 생각은 없고 다른것들은 로직 짤 때 좀 고려를 해봐야 할 것 같다. 꼭 이 자료구조를 써야 하는지 한번 더 생각해보기(특히 Map, 2차원 배열)
